## Objective

Establish the two-column root layout (sidebar slot + main content slot), replace the landing page with an auto-redirect that creates a new chat on every root visit, fix the chat page container so it fills its layout slot correctly instead of claiming the full viewport, and install the `lucide-react` icon package required by Phase 3 components. A non-functional `Sidebar` stub is created so the layout compiles; it is replaced with the full implementation in plan 03-02.

## must_haves

- Root `app/page.tsx` creates a new chat and redirects to `/chat/<id>` with no user interaction required — every visit to `/` lands in a fresh conversation.
- Root layout renders a two-column structure: sidebar column (`w-64 flex-shrink-0`) + main content column (`flex-1 overflow-hidden`), both filling the full viewport height via `h-full flex overflow-hidden` on `<body>`.
- Chat page wrapper uses `h-full` (not `h-screen`) so it fills the `<main>` slot from the root layout without creating a nested scroll conflict.
- Chat page header displays the dynamic conversation title from Postgres (`{chat.title}`) with `flex-shrink-0` so it never gets squeezed.
- `lucide-react` is installed and pinned without `^` in `package.json`, matching the project convention for all dependencies.

Key differences from the scaffold default:
- `body` class: `min-h-full flex flex-col` → `h-full flex overflow-hidden`

  - `flex` (row direction, the default): sidebar and main side by side
  - `overflow-hidden`: prevents double scrollbars; each column manages its own scrolling
- `<Sidebar />` inserted as first child of `<body>` (renders the `<aside>` column)
- `<main className="flex-1 overflow-hidden">` wraps `{children}`:
  - `flex-1`: takes all remaining horizontal space after the 256px sidebar
  - `overflow-hidden`: allows the inner chat page to control its own scroll
- Metadata updated from scaffold defaults to project-specific copy

import { redirect } from 'next/navigation';
import { createChat } from '@/lib/db/queries';

// Auto-creates a new chat on every visit to `/` and redirects immediately.
// The sidebar's "New Chat" button (added in plan 03-03) is the primary creation
// entry point for subsequent chats.

- No `'use server'` directive needed — the entire component is already a Server Component.

The full file after the change:

```tsx
import type { UIMessage } from 'ai';
import { getMessages, getChat } from '@/lib/db/queries';
import { ChatInterface } from '@/components/ChatInterface';
import { notFound } from 'next/navigation';

export default async function ChatPage({
  params,
}: {
  params: Promise<{ chatId: string }>;
}) {
  // IMPORTANT: params is a Promise in Next.js 15+. Must await before accessing.
  const { chatId } = await params;

  // Validate the chat exists — return 404 if chatId is invalid
  const chat = await getChat(chatId);
  if (!chat) notFound();

  // Fetch message history from Postgres (PERS-04)
  const dbMessages = await getMessages(chatId);

  // Convert DB Message[] to UIMessage[] shape expected by useChat's `messages` option.
  // AI SDK v6: UIMessage requires id, role, parts array, and metadata. No top-level `content` field.
  const initialMessages: UIMessage[] = dbMessages.map((msg) => ({
    id: msg.id,
    role: msg.role as 'user' | 'assistant',
    parts: [{ type: 'text' as const, text: msg.content }],
    metadata: {},
  }));

  return (
    <div className="flex flex-col h-full bg-gray-50">
      <header className="border-b border-gray-200 bg-white px-6 py-4 flex-shrink-0">
        <h1 className="text-lg font-semibold text-gray-900">{chat.title}</h1>
      </header>
      <div className="flex-1 overflow-hidden">
        <ChatInterface chatId={chatId} initialMessages={initialMessages} />
      </div>
    </div>
  );
}
```
